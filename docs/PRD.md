# MoneyGoWhere PRD v1

## Summary

MoneyGoWhere v1 is a native iOS app for young professionals to track recurring cashflow through a chat-first workflow and a calendar-led dashboard. The core product is recurring income and recurring costs first, with optional manual entries, per-item reminders, and structured insights that help users understand upcoming obligations and saving opportunities.

The current Figma design covers only the Chat home state and Chat sidebar/history state. There is no dashboard, onboarding, paywall, or manual-entry screen yet, so those must be product-defined first and then designed.

## Product Definition

- Platform: iOS only, greenfield app, native UX.
- Positioning: personal cashflow organizer and recurring-finance coach, not a regulated financial adviser.
- Primary v1 user: single user managing their own subscriptions, bills, salary, and recurring payments.
- Core tabs:
  - `Chat`: conversational logging, follow-up questions, memory-aware assistance, saved chat history.
  - `Dashboard`: month calendar, recurring timeline, item management, reminders, totals, and insights.
- In scope for v1:
  - Onboarding with default currency, locale, and first recurring item.
  - Chat-based logging for recurring income/costs.
  - Manual add/edit from dashboard.
  - Recurring cycles: daily, weekly, monthly, yearly.
  - Optional payment-method label per item, defaulting from profile if set.
  - Per-item reminder toggle with configurable days-before notification.
  - Multi-currency item support with a default account currency and FX-converted totals.
  - Opt-in anonymized aggregate learning for autofill and popularity-based recommendations.
  - RevenueCat subscription with 7-day free trial, `$3.99/month`, discounted yearly plan with final price TBD.
- Explicitly out of scope for v1:
  - Bank sync.
  - Household collaboration.
  - Voice input, image receipts, and attachments.
  - Deep card-management features.
  - Fully free-form financial advice generated by an unrestricted LLM.

## Structured Finance Model

First-class entities:
- `UserProfile`: locale, default currency, notification permission state, default payment method label, aggregate-data consent.
- `RecurringItem`: title, merchant/service, category, icon, amount, original currency, home-currency amount, cadence, next due date, type (`expense` or `income`), payment method label, reminder settings, source (`chat` or `manual`), status.
- `OccurrenceProjection`: generated calendar instances for recurring items.
- `ChatThread`: persisted conversation metadata and messages.
- `ExtractionDraft`: structured fields proposed from chat before confirmation.
- `FinanceMemory`: durable user facts the assistant may reuse, such as salary date, billing date, preferred card, existing subscription names.
- `CatalogEntry`: internal + aggregate-backed defaults for known services, categories, icons, and common prices/plans.
- `AggregateStats`: anonymized opt-in popularity and default-pattern signals only.

Rules:
- Structured records are the source of truth.
- Chat threads are persistent context, but deleting a thread does not delete confirmed finance items.
- Each item stores original currency and a converted home-currency value for rollups.
- Dashboard totals use reference FX rates while preserving original item currency in item detail.

## Chat Tab Behavior

- Empty state follows the Figma chat screen.
- First message intent is recurring finance capture, not general chatting.
- The assistant extracts fields from natural language into a draft record.
- If required fields are missing, the assistant asks targeted follow-up questions.
- The user reviews and confirms before save.
- After save, the assistant confirms what was created and offers the next useful action.
- Chat history sidebar lists prior threads and supports starting a new thread.
- The AI layer should behave as:
  - Deterministic extraction and validation for finance logging.
  - Rules/heuristics for personalized savings nudges and reminders.
  - Optional conversational summarization only if later proven compliant and safe.
- Copy and behavior must avoid regulated-advice framing such as investment, lending, tax, or individualized financial-product recommendations.

## Dashboard Definition

- Top summary: current month income, current month recurring expenses, projected net recurring cashflow.
- Month calendar:
  - Month/year navigation.
  - Dates show projected occurrences.
  - Each date cell can display item icons for due items.
  - Tapping a date opens the item list for that day.
- Recurring items list:
  - Filter by all, expenses, income, upcoming, overdue.
  - Sort by next due date by default.
  - Supports add, edit, pause, delete.
- Item detail/edit sheet:
  - Amount, currency, cadence, next date, category, payment method label, reminder toggle, days-before value.
- Insights section:
  - Rule-based savings opportunities.
  - Duplicate or overlapping subscriptions.
  - Upcoming high-spend periods.
  - Popular alternatives or common subscriptions only from opt-in anonymized aggregate data.
- Manual add flow:
  - Search-first experience with intelligent autocomplete, icons, likely category, and default price/plan where known.
  - User may override any autofilled value.

## Onboarding And Monetization

- Onboarding steps:
  1. Value proposition.
  2. Default currency and locale.
  3. Optional default payment-method label.
  4. Add first recurring item.
  5. Show immediate value preview in dashboard/chat.
  6. Present paywall.
- Paywall behavior:
  - Show after onboarding and first value moment.
  - Trial: 7 days.
  - Monthly: `$3.99`.
  - Yearly: discounted plan, final price TBD.
  - RevenueCat manages offerings, eligibility checks, entitlements, and restore flows.
- After trial expiry without subscription:
  - App remains read-only.
  - Existing records and schedule views remain visible.
  - New item creation, reminder creation, and advanced insights are blocked.

## Data, Privacy, And AI Architecture

- Persistence model: account-backed cloud sync is required for chat history, finance memory, reminders, and subscription state.
- Recommended auth default: Sign in with Apple first, expandable later.
- Aggregate learning is opt-in only and must be anonymized.
- Do not use raw user chat or finance data for cross-user model training by default.
- Keep the assistant architecture schema-first:
  - Parser/extractor.
  - validator.
  - memory updater.
  - recurring projection engine.
  - insights rules engine.
  - catalog lookup/autocomplete service.
- Local-first AI direction for feasibility:
  - Preferred v1 path: deterministic extraction + rules engine + lightweight local ML where needed.
  - If a bundled local model is later used, restrict it to safe assistant phrasing and summarization, not regulated advice decisions.

## Test Plan

- Onboarding creates a default currency, profile, and first recurring item successfully.
- Chat logs a monthly subscription from free text, asks for missing fields, and saves only after confirmation.
- Chat logs recurring income and displays it on the correct future dates in the calendar.
- Manual add/edit from dashboard produces the same structured record shape as chat-created items.
- Reminder settings fire per item using the configured lead time.
- Multi-currency items preserve original currency and roll up correctly into the account default currency.
- Deleting a chat thread leaves confirmed finance items intact.
- Trial-eligible users see correct trial messaging; expired or ineligible users see non-trial pricing.
- Trial expiry transitions the app into read-only mode without data loss.
- Aggregate-driven autofill works only for users who opted in to anonymized collective learning.

## Assumptions And Constraints

- The repo started empty, so implementation assumes a greenfield iOS app.
- The Figma currently defines only chat states; dashboard, onboarding, paywall, and settings need new design work.
- Yearly subscription price is still TBD.
- Household support is not in v1, but the schema should avoid blocking future shared-account support.
- Apple’s Foundation Models framework is on-device, private, and supports extraction, summarization, and tool calling, but it must not be used as the core finance-expert engine for personalized financial guidance as currently framed.
- Local model use on iPhone is technically feasible through custom runtimes, but quality, latency, battery, and app-size limits must be validated before relying on it for primary chat UX.
